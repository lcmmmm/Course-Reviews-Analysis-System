      哪一學年度修課： 110-1

      ψ 授課教師 (若為多人合授請寫開課教師，以方便收錄)
林忠緯

      λ 開課系所與授課對象 (是否為必修或通識課 / 內容是否與某些背景相關) 
資工所選修

      δ 課程大概內容
System Architecture
Timing Analysis
System Design
Advanced Driver-Assistance Systems
Intersection Management
Connectivity
Sensing and Perception
Planning and Control
Verification and Testing
Security
Edge Computing
Certification

      Ω 私心推薦指數(以五分計) ★★★★★
想了解智慧汽車的整個系統架構：★★★★★
想深入了解學怎麼做自駕車系統：★★
涼：★★★★☆  （這半顆星花在期末project要報告也要寫成類論文格式XD）
甜：★★★★★

      η 上課用書(影印講義或是指定教科書)
上課用老師的投影片

      μ 上課方式(投影片、團體討論、老師教學風格)
（疫情影響）這學期是全線上課程，老師表示預先錄影他會重複檢視，
會把不重要或空白的片段剪掉，所以課程節奏較快，每堂課約1-2小時而已。

影片會在上課幾天前就上傳完成，實際上課老師會開一個線上討論空間(自由出席)，
讓同學自由問問題，確實可以感受到老師很認真在授課。

考試也是遠端線上考試，只有一小時，利用gradescope上傳評分，
時間的部分會壓得很緊，避免同學有時間互相討論，也會抓抄襲。

      σ 評分方式(給分甜嗎？是紮實分？)
homework *2 : 20%
quiz     *2 : 30%
project proposol     :  5%
project presentation : 10%
project report       : 35%

偏甜，不會調分，作業不難可以拿滿，考試有一半會跟作業類似，只會有一題比較複雜，
final project的話，基本上會有一個標準分，只要不要沒做分數都不會差太多。

      ρ 考題型式、作業方式
選擇題、表格填充、問答題。
作業也是差不多形式，也是考試會出的基本分。
( 作業會有演算法實作的題目 )
final project可以選擇要做survey、implementation或research，
1-3人一組，根據人數也會對內容深度有不同標準。

      ω 其它(是否注重出席率？如果為外系選修，需先有什麼基礎較好嗎？老師個性？
加簽習慣？嚴禁遲到等…)
不用點名，線上課程自己安排時間即可。
有一點基本的程式能力即可，沒有背景也沒關係，
整個領域的範圍還蠻廣的，上課有聽就可以應付作業考試。

      Ψ 總結
我認為這堂課可以感受得到老師想把所有的關於智慧汽車的內容都帶過一遍，
也很認真在授課，如果想對自駕車、智慧汽車上的架構與應用有初步的認識的話，
我認為很適合從這堂課入門，每個部分都瞭解後，再自己去做深入的學習。
