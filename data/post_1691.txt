      哪一學年度修課：104-2

      ψ 授課教師 (若為多人合授請寫開課教師，以方便收錄)曾建祥

      λ 開課系所與授課對象 材料系大一 必修

      δ 課程大概內容

                Linux基本操作
                Python 安裝與基礎 數字型態 序列 運算
                Python 流程控制
                模組與函式
                Python 輸入輸出字串處理
                Python 錯誤與例外處理
                Python 時間函式
                Python 數學模組與繪圖模組
                Python 模組

      Ω 私心推薦指數(以五分計) ★ 原本還有兩顆星，原因後述

      η 上課用書(影印講義或是指定教科書) 有參考用書但是基本上都用講義

      μ 上課方式(投影片、團體討論、老師教學風格)

                就課程大綱來講沒有大問題，該講的都有講到，但是老師的表達可能跟

                大家的理解有點落差，導致吸收的狀況一直都不太好(個人主觀認定)

                此外，上課實際操作的部分非常的少，常常一路講下去，後來去修別系

                的計程，才深刻理解邊講邊操作、改參數去思考原理等等，對學基本程

                式的重要性，修過一個課程安排完整的計程後回來看，真的會對於這門

                課有相當大的疑問(一顆星扣在這裡)

                以上可能是比較主觀的部分，在課程編排的順序上個人也有點疑問，常

                常是作業都交了才開始上那個部分的課程(ex時間函式、字串處理)，所

                以很多Python的相關知識我們都是從網路上自行google的

      σ 評分方式(給分甜嗎？是紮實分？)

                HW 10%(2%,4%,4%)

                project 50%(期中報告10%，期末報告40%)

                課程參與5%

                期末考35%

                整體來講紮實偏甜，project 84.4 期末考91 最後拿A，但某種程度上

                有點無言(後述)

      ρ 考題型式、作業方式

                作業第一份是基本名詞的查詢，算是入門

                第二份要從老師的資料庫擷取降雨量並且分析，這個我有相當大的疑問

                在於老師上課對作業用到的技巧大概有一半以上沒有提到，除了前述

                的課程順序，擷取資料對沒有基礎的同學相對吃力，也是整份作業最

                關鍵的一步，但是這部分在作業說明還有上課講義都沒什麼提到，幾乎

                也是大家從頭摸索

                第三份作業相對就課堂中有提到，寫起來就輕鬆很多，整體來講還是會

                學到基本的資料庫整理

                project有50%，但是大家做的難度差異很大，有些人做了遊戲，我們這

                組做的是四根柱子的河內塔最佳解，算是網路上查一下，約出來一次就

                可以整個搞定的東西，最後可以拿84分真的有點無言，雖然說是CP值很

                高，我們也過很爽，但是完全沒有學到東西，當然想學是可以挑更難的

                題目，但課程安排上也沒有做這樣的引導

                很多人選擇遊戲，但是沒有方向困擾很久，因為要從頭學起，修過的另

                一門計程至少有教基本的SDL使用，可以有個方向然後看youtube學自己

                需要的部分，同為程式入門，兩個project投資的時間和學到的差很多

                期末考基本上是投影片看懂就可以進去考，手寫幾個程式碼，解釋幾個

                名詞就會有不錯的分數，算是比較沒有壓力

      ω 其它(是否注重出席率？如果為外系選修，需先有什麼基礎較好嗎？老師個性？
加簽習慣？嚴禁遲到等…)

                出席率基本上不太點名，程式語言入門所以不需要什麼基礎，但如果是

                外系，強烈建議去修別系的計程，加簽基本上以教室坐滿為原則

      Ψ 總結

                雖然說程式設計對於每個系的重要度各異，教的內容可能深淺有差，

                loading的也不一樣，但是我認為一門課幾個基本的點都很有疑問，比如

                前述的作業與上課內容幾乎平行時空、對於50%的project沒有範例也沒

                有引導，最後交個河內塔就跟熬夜寫遊戲的人分數差不多，這門課可以

                設定成只要入門會寫，也可以設定成要project或資料處理達到一定的水

                平，但最後三學分學了一大堆自己估狗都比較清楚的東西而且很雜，感

                覺不到課程安排的主軸，修過其他系計程後，更是加深這個感覺
