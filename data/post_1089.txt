      哪一學年度修課：107-1
ψ 授課教師 (若為多人合授請寫開課教師，以方便收錄) 楊烽正 
λ 開課系所與授課對象 (是否為必修或通識課 / 內容是否與某些背景相關) 
機械系必修 
δ 課程大概內容
Quick start for C# programming(Compiler)
Quick start for C# Windows programming
Data types and syntax
Expressions and operators
Statements
Array Type
Methods and Arguments
Enum,Struct
Class
Sample applications
Ω 私心推薦指數(以五分計) 
☆☆☆☆☆(對 零顆的意思)

       本身擅長C#想耍廢拿高分：★★★★★
       完全不具備程式基礎：☆☆☆☆☆
η 上課用書(影印講義或是指定教科書)
無，老師會公告投影片在Ceiba
μ 上課方式(投影片、團體討論、老師教學風格)
        上課內容的各個主題都會有課程影片，內容算是計算機概論會教的東西，每週大
        約3小時的進度，自己在家看，不過內容基本上就是看老師照唸投影片。
        上課時間老師會先介紹當週主題，再帶大家做，但是個人認為他的表達能力不是
        很好，有些概念聽會的同學講都比他清楚很多。基本上一週一個主題，如果他發
        現課上不完就會開始講超快，同學就可以在完全搞不清楚自己在做甚麼的情況下
        學會快速地打字以跟上老師的進度囉！如果直到打鐘老師依然沒辦法把課上完，
        他會再次加速，總之一定要在當週把主題結束。不過通常上到這裡，也就是打鐘
        後大概30分鐘左右，老師的程式就會出現bug，然後在全班50人左右看著他debug
        de不出來的情況下，老師就會放全班下課，然後再補影片把剩下的東西講完，就
        算晚下課加上補充影片的時間都到一節課了(而且是在講超快的情況下)，依然不
        願意把主題分成兩週，算是相當特別的堅持呢！
σ 評分方式(給分甜嗎？是紮實分？)
        課後作業 30%
        課堂小考 10%
        期中考 30%
        期末考 30%

        算紮實甜吧(？
        本身小考分數6-7分左右(滿分10)，期中76期末60幾，作業都有準時交(功能完整)
        ，拿A-
        老師一直強調多寫幾行註解助教會給比較高分，我沒多寫甚麼註解所以不知道。
        但是作業只要功能完整分數都有100，小bug扣分不會很重。
ρ 考題型式、作業方式 
        第一節課有隨堂測驗，如果遲到太久可能考不到。滿分10分，以課程影片內容為
        主。原本兩週考一次，但是老師發現都沒有人在看他的影片之後就變成每週都要
        考。可能大家覺得3小時的課後影片考出來只占10%，CP值不高，所以大家看影片
        的狀況不是很理想，就會反映在小考成績上。
        作業的部分通常都不難，就是上課內容的延伸，所以上課內容有跟上的話很快就
        可以寫完了。但如果碰到老師快教不完然後上超快，或是有些較複雜的、老師講
        不清楚的程式邏輯，那就另當別論xD
        期中期末考型式相同，皆為手寫，20分的是非題考計概，20分修正錯誤程式碼，
        剩下是回答程式回傳值跟手寫程式。
ω 其它(是否注重出席率？如果為外系選修，需先有什麼基礎較好嗎？老師個性？
加簽習慣？嚴禁遲到等…)
        老師還有一個特色，第一節課開始，老師就經常重複一句「我不可能從最基礎的
        一步一步教」，所以毫無程式基礎的同學就得把用到但還沒教過的內容背起來。
        有完全沒學過程式的朋友每個禮拜寫作業都要自己查資料，找網路上的C#教學來
        看，每個禮拜花在計程的時間上超多，期中考之後就停修了。
Ψ 總結 
        如果想被程式電，就去修資工
        想成為程式大神去修電機
        想不開再來修楊教授的計程= =
