      哪一學年度修課： 107-1

      ψ 授課教師 (若為多人合授請寫開課教師，以方便收錄)

      陳炳宇+王銓彰

      λ 開課系所與授課對象 (是否為必修或通識課 / 內容是否與某些背景相關) 

      資訊系及資管系選修

      δ 課程大概內容

      1. Introduction of the Course and Rendering Pipeline
      2. Introduction of Game Project
      3. Introduction of Unity
      4. Game System Analysis
      5. Game Control System
      6. Game Mathematics
      7. Game Geometry
      8. Lighting & Texturing
      9. Game AI
      10. Game FX
      11. Game Physics

      Ω 私心推薦指數(以五分計) ★★★★★

      想認真做遊戲 ★★★★★
      (Unity 基本靠自己讀，沒有啃下去的決心可能會有點辛苦)
      想混分數 ★ ~ ★★★★★
      (看組員，但最好還是不要)

      η 上課用書(影印講義或是指定教科書)

      沒有指定用書，老師網站有ppt及參考書

      μ 上課方式(投影片、團體討論、老師教學風格)

      以投影片為主
      有小組報告課(Check Point)跟討論課(學期初一堂，Brainstorming)
      (Check Point 會檢查進度並由陳炳宇老師和業師(Ta Na)給階段評語)

      小組討論基本上要自己約時間或是利用下課時間
      (期初分組的時候也有要同學討論讓想做比較接近的東西的人同組)

      課程部分陳炳宇老師上的主要是 Game Geometry 跟 Lighting & Texturing
      剩下的部分大多是業師(王銓彰老師)上課
      然後 Unity 大部分要自己學...

      陳炳宇老師上課的部分基本是計算機圖形課，雖然跟一學期的計圖比起來較淺
      而兩位業師的課程主要是策劃相關的東西，還可以聽一些經驗跟八卦

      σ 評分方式(給分甜嗎？是紮實分？)

      這學期把作業跟期中拔掉了，變成 100% project
      而 project 的部分助教是說為前面的 Check Point 報告分數
      加上最後的 Demo 分數
      個人覺得分數偏甜，25 組裡大約 9 組 A+、10 組 A
      (目前的分數)

      ρ 考題型式、作業方式

      100% project，最後 Demo 跟每次的小報告都算分。
      最後是直接給一個等第，中間的分數聽停修的同學說可以找助教看
      (他們是因為中間看到的成績不好看停修的...)

      每次報告都有報告時間跟Q&A時間

      ω 其它(是否注重出席率？如果為外系選修，需先有什麼基礎較好嗎？老師個性？
加簽習慣？嚴禁遲到等…)

      最重要的大概就是要會寫程式，要會找資料解決問題
      Unity 主要是 C#，如果想選課但不會寫程式的可以提前去找各種教學
      如果有程式基礎，課選上再開始看 Unity 也來的及
      (主要還是每組人不多，再加上程式其實會花不少時間，
      最好還是大家都會做會比較方便)

      策劃的想法也有點重要，不過這個我覺得見仁見智
      因為基本上是分組製作，討論的過程就會有不少想法出來

      美術大部分的組是用網路素材(包含 Unity Asset Store)

      Ψ 總結

      會來發評價文主要是想幫這門課平反一下
      之前原本有想選但看到板上的評價文很怕整個不敢選
      後來是聽到去年修課的同學說改用 Unity 才敢選下去
      (一開始就有 Unity 要自己學的心理準備，後來果然是這樣XD)
      (但 Unity 真的不難，跟有的資工系選修寫作業像在鬼打牆一樣比起來好很多)

      要注意的是選這門課找組員跟想策劃很重要。
      分數高的組大部分的策劃都做的不錯，
      甚至比一些遊戲做的更完整但策劃沒那麼好的組分數高。

      然後基本上之前那篇評價文說的 mailing list 的問題有改進
      期初這學期原本也是要用 mailing list(有在課程網站公佈並上課公佈要記得加)
      但 mailing list 炸了，所以是用 FB 社團的方式。

      在 Spec 的部分，今年修的時候感覺都講的還算清楚。
      沒有限制 Game Engine (雖然推薦 Unity，但也有組用 RPG Maker，
      也有說你要用 Unreal Engine 也可以)
      (Check Point 跟 Final 投影片要放什麼內容沒有講的很清楚... 但問題不大)

      這門課其實真的沒有板上之前文章說的那麼雷
      如果對做遊戲有興趣而且肯花不少時間下去做的話還是蠻推薦修修看的

      (我們這組有不少東西幾乎是最後一兩個月才開始做，但最後還是有A+，
      雖然我最後一個月我花在這個 project 的時間破百就是了...(做美術+各種場景+
      程式))(主要是因為我們這組只有兩個人，人多應該會相對比較輕鬆)

      (另一個組員雖然花的時間比較少，但他也說是他這段時間 Loading 最重的一門課)
